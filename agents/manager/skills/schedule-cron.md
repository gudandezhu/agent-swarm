# 技能：调度定时任务

## 功能描述

当用户要求设置定时任务时，你需要将用户的自然语言描述转换为标准的 cron 配置格式。

## 工作流程

1. **理解用户需求**：识别用户想要定时执行的任务
2. **生成配置**：将自然语言转换为 cron 配置
3. **返回 JSON**：严格按照格式返回配置

## 输出格式

你必须返回以下 JSON 格式（不要有其他内容）：

```json
{
  "type": "schedule",
  "config": {
    "agentId": "manager",
    "schedule": "cron表达式或毫秒数",
    "task": "任务描述（原用户的自然语言）",
    "handler": "处理技能名称（可选）",
    "timezone": "Asia/Shanghai"
  }
}
```

## Cron 表达式规则

标准格式：`分 时 日 月 周`

| 字段 | 值范围 | 特殊字符 |
|------|--------|----------|
| 分 | 0-59 | `*` `,` `-` `/` |
| 时 | 0-23 | `*` `,` `-` `/` |
| 日 | 1-31 | `*` `,` `-` `/` |
| 月 | 1-12 | `*` `,` `-` `/` |
| 周 | 0-7 (0和7都是周日) | `*` `,` `-` `/` |

## 常见示例

### 每天定时

| 用户输入 | Schedule |
|---------|----------|
| "每天早上9点" | `0 9 * * *` |
| "每天晚上10点" | `0 22 * * *` |
| "每天中午12点" | `0 12 * * *` |
| "每天凌晨3点" | `0 3 * * *` |

### 间隔定时

| 用户输入 | Schedule (毫秒) |
|---------|----------------|
| "每30分钟" | `1800000` |
| "每1小时" | `3600000` |
| "每2小时" | `7200000` |

### 每周定时

| 用户输入 | Schedule |
|---------|----------|
| "每周一早上9点" | `0 9 * * 1` |
| "每周五下午5点" | `0 17 * * 5` |
| "每周三中午12点" | `0 12 * * 3` |

## Handler 类型

对于不同类型的任务，使用相应的 handler：

| 任务类型 | Handler |
|---------|---------|
| 汇报工作 | `report` |
| 发送提醒 | `reminder` |
| 检查任务 | `check-tasks` |
| 清理数据 | `cleanup` |

如果不确定，可以不指定 handler。

## 示例对话

### 示例 1：每天汇报

**用户**：每天早上9点提醒我汇报工作

**返回**：
```json
{
  "type": "schedule",
  "config": {
    "agentId": "manager",
    "schedule": "0 9 * * *",
    "task": "每天早上9点提醒我汇报工作",
    "handler": "report",
    "timezone": "Asia/Shanghai"
  }
}
```

### 示例 2：间隔检查

**用户**：每30分钟检查一次任务

**返回**：
```json
{
  "type": "schedule",
  "config": {
    "agentId": "manager",
    "schedule": 1800000,
    "task": "每30分钟检查一次任务",
    "handler": "check-tasks",
    "timezone": "Asia/Shanghai"
  }
}
```

### 示例 3：每周提醒

**用户**：每周一早上8点提醒我开例会

**返回**：
```json
{
  "type": "schedule",
  "config": {
    "agentId": "manager",
    "schedule": "0 8 * * 1",
    "task": "每周一早上8点提醒我开例会",

```

## 注意事项

1. **只返回 JSON**：不要添加任何解释性文字
2. **schedule 类型**：
   - 使用字符串表示 cron 表达式
   - 使用数字表示毫秒间隔
3. **task 字段**：保留用户的原始描述
4. **时区**：默认使用 `Asia/Shanghai`
5. **agentId**：通常使用 "manager"，除非用户明确指定其他 Agent

## 错误处理

如果无法理解用户的需求，返回：

```json
{
  "type": "error",
  "message": "无法理解定时任务需求，请更具体地描述时间和任务内容"
}
```
